OCADStringEntry* ocad_string_entry_new ( OCADFile *  pfile,
u32  size 
)

Creates a new string entry large enough to fit the specified number of bytes.

If there is an empty string index entry of sufficient size, it will be used; otherwise, a new entry will be allocated. The entry returned will have its ptr and size fields set; the size field may be larger than the size requested in the parameter. The caller is responsible for copying data into the actual string (retrieved via ocad_string()). If there is not enough memory to create a new entry, NULL is returned.
